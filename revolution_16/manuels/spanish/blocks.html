Bloques

NPDS, permite la visualización de un bloque (o de varios) sobre las páginas.
NPDS (con un tema conveniente) permite indicar estos bloques sobre casi todas las páginas.

Un Bloque puede ser reservado a los administradores, a los miembros, a N grupos de miembros, a los Anónimos o ser visible por todos.

Modificar un bloque

Cambie la información necesaria en el bloque y seleccione la acción deseada, para terminar presione el botón OK.
Solo puede modificar un único bloque a la vez.

Borrar un bloque

Seleccione el bloque y eliga la acción borrar un bloque, para terminar presione el botón OK.

Crear un bloque (de izquierda o derecha)

Escoja el título del bloque, su índice (lo que va a permitir una visualización clasificada así como su contenido y presione el botón crear un bloque.... Por defecto el bloque recientemente creado sera activo.


Bloque y Tampon memorizado

- los bloques pueden "imprimirse en memoria tampon " y pueden ser activados o desactivados sin necesidad de tocar el tema actual
- No se pondra en memoria un bloque con un tiempo de máscara de 0 (cero).
- dispone también de una ayuda en línea que quizá utilizara mediante el módulo 'ayuda-del contexto' o en el cuadro de un desarrollo PHP (dejar esta zona vacía si no).

NPDS dispone de un sistema de bloques ancho capaz de indicar o ejecutar mucho mas:
- HTML (para un bloque que soporta el HTML, seleccionar la opción HTML en el momento de su creación)
- Javascript
- Código PHP y/o SSI (Server Side Include)
- Función PHP y/o de NPDS

La única posibilidad de incluir un código sera respetando la siguiente sintaxis:
. Para el código PHP y/o SSI:
include#su_carpeta/su_fichero.php (sin ', sin").
Este fichero debe ser en php y debe devolver obligatoriamente $content.

. Para Funciones PHP y/o funciones de NPDS:
function#su_funcion = function#Xblock (sin ', sin ", sin () y sin;).

Si su función dispone de parámetros:
function#su_funcion= function#XNews (sin ', sin ", sin () y sin;). params#sus_param = params#5,10,false (sin ', sin" pero separados por comas).

por ejemplo: function#category

function#oldNews params#$storynum

function#RecentForumPosts
params#Forums Infos,15,0,false,10,false,-: 
Puede también poner las 2 fases:
 include#su_carpeta/su_fichero.php (Sin ' ").
 despues
 function#su_funcion => function#XNews (Sin ' " () ;).
 params#sus_param => params#5,10,false (Sin ' " pero separados por comas).
 => ¡Esto le permite cargar un código y ejecutarlo!

Bloque CSS:
class-title#nombre de una clase en su fichero CSS.
class-content#nombre de una clase en su fichero CSS. = Estas líneas permiten asociar para CADA BLOQUE un CSS (título y contenido)!
- Theme-Dynamic dispone de dos nuevas palabras meta: B_class_title y B_class_content que pueden ser utilizadas en el código de bloc.html

- Los temas PHP disponen de dos nuevas variables: $B_class_title y $B_class_content que puede 'ser importadas' en la función themesidebox (mediante una global...)

    Bloque URI

     uri#uris_separadas_por_un_espacio no indicar el bloque que sobre las URI citadas.
     El tiempo de retención debe ser igual a 0 (cero) en este caso.
     Ejemplo: uri#index.php user.php forum.php static.php?op=statik.txt&npds=1
     El tag uri# permite el uso del comodín *: por ejemplo: viewforum.php?forum=* o viewtopic.php?topic=*&forum=10
     => Esta sintaxis es la sola soportada!

    Bloque Hidden

     hidden#... para no mostrar el tema de los bloques, pero la carga del contenido.
     Esto permite utilizar el sistema de bloqueo de "carga" de Javascript, los archivos PHP, CSS, ...
     ejemplo : hidden#<script type="text/javascript" src="lib/yui/build/yui/yui-min.js"></script>
     => Esta sintaxis es la sola soportada!
