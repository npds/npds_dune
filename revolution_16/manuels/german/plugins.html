Admin Plugins


Hier können Sie die Plugins für NPDS administrieren.

Der Modulersteller (zum Beispiel IPBAN) tut sein Bestes um die Konfiguration der Pugins in NPDS zu gewährleisten.

Nach der Installation eines Moduls mit eigener Konfiguration(via Plugins), erscheint das Modul in der Liste.

Die Beschreibung eines ADmin Plugins ist (file : admin/extend-modules.txt):
Für jedes Plugin :
[module]
    [nom]Plugin Name[/nom]
    [ModPath]Pfad zum Plugin (i.e. : modules/ipban)[/ModPath]
    [ModStart]die Datei die das Plugin startet (i.e. : ipban)[/ModStart]
    [niveau][/niveau]
[/module]
.
.

Tags : [niveau][/niveau] ist optional (oder leer) : in diesem Fall können nur SuperAdmins das Plugin konfigurieren.

Tags : [niveau][/niveau] integriert EINES (EINS meint 1) der folgenden Werte, nur SuperAdmins UND Administratoren dieser Funktion haben Zugang.
 - Werte :
     - radminarticle (Articles and Publish),
     - radmintopic (Topic),
     - radminleft (Left blocks),
     - radminmain (Main block),
     - radminright (Right blocks),
     - radminsurvey (Poll),
     - radminsection (Sections),
     - radminlink (Links),
     - radminephem (Ephemerids),
     - radminhead (Headlines),
     - radminfaq (FAQ),
     - radmindownload (Downlaod),
     - radminforum (Forums),
     - radminreviews (Reviews),
     - radminlnl (Newsletter),
     - radminsdv (Groups management),
     - radminuser (Edit Users),
     - radminfilem (FileManager),
     - radminsuper (Super Admin)
     - no-right (All Admins even if admin have NO Right)

Zum Beispiel :
- Ein Plugin das [niveau]radminlnl[/niveau] als 'level' hat wird angezeigt für SuperAdmins UND Newsletter Administratoren.

- Ein Plugin das [niveau]no-right[/niveau] als 'level' hat wird für alle Administratoren angezeigt, ausser der Moderator hat eingeschränkte Rechte.
