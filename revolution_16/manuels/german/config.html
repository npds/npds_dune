Settings


Wenn Sie damit beginnen NPDS zu nutzen, ist das die wichtigste Funktion. Sie können die wichtigen Informationen einstellen und das Aussehen Ihrer Seite beeinflussen.

Folgend die Schlüsselfunktionen von NPDS

Generelle Seiteninformationen

  • Parse algo : Sie können eine der zwei Parse Optionen aussuchen.
    - FixQuotes ist die 'Standardeinstellung' Option, welche die ['] doppelt (z.B. : l'art wird l''art)
    - StripSlashes erlauben dieses Problem zu verhindern (i.e. : l'art bleibt l'art) es funktioniert aber nicht immer.


  • OB_Gzhandler : Erlaubt die Verwendung von 'compressed data' in geeigneten Browsern.
    Bevor Sie komprimierte Daten verwenden, überprüfen Sie welche Kompression von den Browsern unterstützt wird ("gzip", "deflate" oder keine) dann wird es zum Bufferinhalt zurück gesendet je nach Haederinformation.

    Arbeitet nur unter Php 4.05 (oder >) + Aktivem Zlib (perform a phpinfo();) und aktiviertem mod_gzip für Apache.


  • DateiManager : Erlaubt, oder verbietet den FileManager Zugriff. Warnung! Nutzen Sie es nur, wenn Sie es wirklich brauchen


  • Cookies TTL (Admin und User) : Kein Kommentar (1 year = 8760 hours)


  • Seitenname : Ihre Seite hat einen Namen, hier wird er eingegeben!


  • HTML Seitenname: HTML Seitentitel (hilfreich für Suchmaschinen)


  • Seiten URL : http://www.xxx.com or http://www.xxx.com/homepage/...


  • Seiten Logo : .gif oder .jpg Datei, gespeichert im images Ordner auf dem Server (Standard) oder in einem Ihrer Templates (voller Pfad: themes/my_theme/..../logo.jpg zum Beispiel). Diese Datei wird benötigt für die NPDS Druckfunktion. Wir empfehlen ein Schwarzweiss Bild.

  • Seiten Slogan : Wenn Ihre Seite einen Slogan hat, kommt er hier hinein!


  • Seiten Start Datum: Ein Datum (20/10/1429 oder 31ter Dezember 2012) wird nur in Statistiken angezeigt.


  • Anzahl der Artikel in TopPage : Erlaubt die Einstellung der Top x (5, 10, 15, ...).


  • Anzahl der Artikel auf der Seite : Wieviel Artikel sollen auf der Seite angezeigt werden? 10 ist ein guter Wert, es hängt aber vom Inhalt Ihrer Seite ab. Es ist nur ein Vorschlag, es liegt bei Ihnen.

  • Artikel in der Old Articles Box : In den rechten Boxen/Blöcken, finden Sie die Old Articles. Diese Option erlaubt es Ihnen, die Anzahl der angezeigten alten Artikel einzustellen.


  • Export-News aktivieren? : Export-News sind zwei Textdateien, die sich im Seitencache befinden. Standardeinstellung ist = Nein.
    In diesen beiden Dateien (ultramode.txt and net2zone.txt) sind die letzten 10 Artikel der Seite gespeichert.
    Diese Dateien können Inforamationen von anderen Seiten enthalten, unter Verwendung einfacher Strukturen. :
    -ultramode.txt ist eine Textdatei in der Artikel getrennt werden durch %%
    -net2zone.txt ist eine XML Text Datei für net2zone.

    Die "Online" Funktion die genau so arbeitet ist Backend welches das RDF/XML Dateiformat oder besser unterstützt, das NPDS Push Infos System Module.


  • Anonymous Default Name : Nicht registrierte Besucher werden Anonymous genannt. Anonymous können Gäste, Besucher, ... sein. Ändern Sie diese Einstellung nicht.


  • Allow the post of News : Erlaubt das Erstellen von Artikeln (oder nicht) für Moderatoren / Administratoren oder Mitglieder.
    Das ist wichtig, wenn Sie eine Informationsseite unterhalten.


  • Admin Klicks nicht in die Satistik speichern : Das braucht keine Erklärung.


  • Default Template Ihrer Seite : Wählen Sie das Default Template. Behalten Sie im Hinterkopf, das Benutzer ein eigenes Template für Ihr Benutzerpanel verwenden können (wenn mehr als ein Template vorhanden ist...).

  • Start Seite : Diese Funktion erlaubt die Einstellung der Startseite (Home). Standard ist index.php Sie können aber auch eine andere Seite einstellen zum Beispiel :
    - links.php
    - faqs.php
    - static.php?op=npds.txt&npds=1 ... (npds kann sein -1 : keine Boxes/Blöcke, 0 : left box/block, 1 : linke und rechte Boxen/Blöcke)

    - EDITO :
    => index.php?op=edito : zeigt Edito UND die News an (ohne "Nächste Seite" Link)
    => index.php?op=edito-nonews : zeigt Edito OHNE die News an (mit "Nächste Seite" Link)
    => index.php?op=newindex : zeigt News OHNE Edito (mit "Nächste Seite" Link)
    => index.php?op=edito-newindex : zeigt Edito UND die News an (mit "Nächste Seite" Link)
    => index.php?op=newtopic&topic=X : zeigt News im Zusammenhang mit Thema'id X OHNE Edito (mit "Nächste Seite" Link)


  • Wählen Sie die Sprache für Ihre Seite : Kein Kommentar.


  • Multi-language Fähigkeit : Erlaubt NPDS die Multi-Language Funktionen mit Tags [language]...[/language].
    Vergessen Sie nicht das lokale Format einzustellen (wie : [french]fr_FR[/french][english]en_EN[/english]).


  • Lokales Zeit Format : Hängt von Ihrer lokalen Zeitzone und Unix von den Funktionen in /usr/share/locale ab


  • Tag ist ab UND Nacht wird es ab : Genutzt von EDITO (oder anderen Funktionen) zum Ändern der eingestellten Zeit


  • GMT : Abhängig (+ oder -) von der Einstellung auf Ihrem Server.

    • Banner Optionen

  • Activieren Sie Banner auf der Seite : Wollen Sie Banner erlauben? Ja oder Nein. Wenn Sie Ja wählen müssen Sie zunächst die Bannereinstellungen konfigurieren...


  • Ihre IP für unregistrierte Klicks : Ihre IP muss hier eingestellt werden, damit Sie eine ehrliche Statistik erhalten.

    • Footer Informationen

  • Footer Zeile 1,2,3,4 : Footer Zeilen sind HTML-freundlich. Wenn Sie nur 1 or 2 Zeilen brauchen, lassen Sie einige Felder leer (beginnend vom Ende).
    Hier können Sie Infarmationen zu Ihrer Webseite und das Copyright einfügen, ...

    • Backend Konfiguration (RDF/XML News Känäle)

  • Backend Titel : Sie können Ihren Slogan direkt hinter dem Titel eingeben

  • Backend Language : Siehe Sprache Ihrer Seite.


  • Backend Image URL : Voller Pfad (http://...) passend zu Ihrem Webseiten Slogan.


  • Backend Image Breite : Kein Kommentar.


  • Backend Image Höhe : Kein Kommentar.

    • Web Link Standardeinstellung

  • Links pro Seite : Maximale Anzahl der Links pro Seite (Weblink Optionen).


  • Hits um populär zu werden : Kein Kommentar. Wenn aktiviert, wird ein kleiner Text zu einem neuen Link angezeigt.


  • Anzahl der Links die als neu angezeigt werden : Maximale Anzahl der Links im "Neu" Bereich.


  • Anzahl der meistbesuchten Links : Maximale Anzahl der Links im meistbesuchte Links Bereich (Beispiel: die 10 meistbesuchten Links).


  • Anzahl der Links die in der Suche angezeigt werden sollen : Kein Kommentar.


  • Sollen anonyme Besucher Links einsenden können : Ja oder Nein


  • Logos für die Hauptweblinks aktivieren : Ja oder Nein.


  • Sollen Icons für neue Kategorien angezeigt werden? : Ja oder Nein? Wenn Ja, wird ein Icon für die neue Kategorie, oder wenn es neue Links in einer kategorie gibt angezeigt.

    • Mail System (Email)

  • Administrator E-mail : Wird genutzt, wenn NPDS E-mails versendet und erlaubt es der Person, an die die E-mail gesendet wurde zu antworten (Echte oder Fake E-mail / mit dem Namen der Domain => z.B. : developpeur@npds.org).


  • Welche E-mail Funktion soll genutzt werden ? Funktion : mail =>Standardeinstellungen für Linux Server / Funktion : email => Für andere Server Systeme.


  • Footer der Email : NPDS sendet E-mails (beispielsweise ein Passwort bei Neuanmeldung) und diese Option erlaubt Ihnen die Gestaltung des Footers der E-mails. Es ist möglich HTML Code zu verwenden. Sie sollten auch Ihren Namen und Ihre Webadresse mitsenden..
    Mail bei neuen Einsendungen an den Admin. NPDS erlaubt es eine E-mail an den Admin zu senden, wenn es neue Einsendungen für die Webseite gibt (Artikel, Links, Forum, usw).

  • Informieren bei neuen Einsendungen : Ja oder Nein.


  • E-mail Adresse für die Information : Kein Kommentar.


  • E-mail Betreff : Kein Kommentar.


  • E-mail Inhalt (Mitteilung) : Kein Kommentar.


  • E-mail Adresse (von) : Kein Kommentar.

    • Kommentar Optionen

  • Art der Moderation : Das funktioniert, aber vertauen Sie nicht zu sehr darauf...

    • - Moderation durch Admin
    - Moderation durch Benutzer
    - Keine Moderation

  • Erlaubt Anonymous Kommentare zu schreiben : Ja oder Nein.

  • Anzahl der Kommentare pro Benutzer und pro Tag (24 Stunden) : Liomitiert die Anzahl der Kommentare pro Benutzer und Tag. 6 ist Standard, 99999 ist das Maximum.

    • Umfrage Optionen

  • Maximale Anzahl der Umfragen :


  • Erlaubt es Benutzern mehrfach abzustimmen : Ja oder Nein, es liegt bei Ihnen. Wenn Sie 'Ja' angeben, Können Benutzer einmal am Tag abstimmen und einen Kommentar schreiben. Wenn Sie 'Nein' angeben, bleibt es bei einer Stimme, solange die Umfrage aktiv ist.


  • Kommentare in Umfragen Zulassen : Ja oder Nein.

    • Einiges zu Icons und Grafiken

    Beispiele für das anlegen von Ordnern für Bilder ... .

  • Pfad zu Themen Bildern : Brauchbar wäre /images/themen.


  • Ein anderer Bildpfad (Umfrage, ...) : Brauchbar wäre /images/umfrage/.


  • Grafiken im Administrationsmenu : Brauchbar wäre images/admin/.


  • Grafiken im Adminmenu: Ja oder Nein.


  • Kleines Administrationsmenu : Ja oder Nein.


  • Schriftarten für die Seite : Brauchbar sind: Verdana,Arial,Helvetica. Sie können es ändern, aber denken Sie an das Gesamtbild Ihrer Seite...

    • Diverses

  • Anzahl der Artikel, die im Adminbereich angezeigt werden sollen Admin : Die Anzahl gilt auch für die Anzeige der Abschnitte.


  • Minimale Länge für Passworte : 8 Buchstaben ist ein guter Wert.


  • Anzahl der Benutzer, die auf einer Seite in der Liste angezeigt werden sollen:


  • Avatare verwenden : Sollen Avatare für in Benutzerprofilen, im Forum, verwendet werden, oder nicht?


  • Maximale Grösse der Avatare in Pixeln (Breite * Höhe / 60*80) : Sie können die maximalle Grösse für den Upload von Avataren einstellen.


  • Kurze Benutzerregistrierung : Dabei wird es dem neuen Benutzer nicht erlaubt, seine ICQ, MSN, AIM Nicknamen einzutragen.


  • Erlaube automatische Konfiguration für neue Benutzer : NPDS allows a new member to wait until an admin validates his profile (value : allow the connexion). Usefull for "closed" portals.


  • Erlaube dem Benutzer, sein Passwort auszusuchen : Sie können den Benutzern erlauben ein eigenes Passwort auszusuchen. Sie können es aber auch verbieten und so die Benutzerrechte einschränken


  • Anmeldung erlauben : Erlaubt es Benutzern sich anzumelden oder nicht. Das entscheiden Sie


  • Unsichtbare Benutzer erlauben : Erlaubt es Benutzern für andere Benutzer 'unsichtbar' zu bleiben (nur Administratoren können sie dann sehen). Kann interessant sein für Moderatoren im Forum.

  • Anmeldung schliessen : NPDS erlaubt es die Anmeldung für eine bestimmte Zeit zu schliessen. (die static/closed.txt Seite wird angezeigt.


  • HTTP Referers aktivieren :


  • Wie viele Referer sollen erlaubt sein : Erlauben Sie nicht zu viele, das füllt nur die Datenbank!


  • Benutzer Liste : Privat : NPDS verbietet anonymen Besuchern, die Benutzerliste einzusehen.


  • Download Bereich : Wählen Sie die Kategorie, welche als erste im Downloadbereich angezeigt werden soll.

    • HTTP Diverses

  • Überschriften : Kontrollieren Sie ob der Port 80 für das Web frei ist :


  • Cache Kontrolle : Aktiviert die erweiterten Cache Tags (pragma ...) : NPDS ermöglicht es, die erweiterten Cachemanagement Funktionen zu nutzen. Einige Hoster unterstützen diese Funktion aber nicht.


  • DNS Auflösung; aktiviert für Einsendungen (Forum, ...) : NPDS ist es möglich, die IP Adresse und die DNS in Echtzeit aufzulösen. Das kostet allerdings Recourcen und einige Hoster verbieten es auch

    • SYSTEM Diverses

  • Maximale Dateigrösse für SaveMysql : Maximale Dateigrösse des ASCII SaveMysql Dump vor der Kompression.


  • SaveMysql Datei : Art der Datei für den SaveMysql Dump (beispielsweise gZip).


  • Tiny-Mce Editor aktivieren : NPDS kann den Tiny-Mce Editor (alle Browser).
