管理 - 首选项



本功能是您开始创建一个NPDS网站时最重要的部分。您可以修改您网站中的所有重要信息，尤其是您期望的外观效果。

您在以下的段落中将找到重要参数的解释，尤其是在NPDS中使用的新功能。


网站的简介


  • Parse algo : 对于您网站上的参数，您可以选择Parse的一个或另一个功能。
    - FixQuotes 是原功能，可以将一个单引号自动变为一个双引号 (例如 : l'art 变为 l''art)
    - StripSlashes 是为了保证不影响文本本身 (例如 : l'art 仍然为 l'art) 但是可能会产生其它问题。


  • OB_Gzhandler : 允许向浏览器发送压缩数据，当然，该浏览器必须支持压缩页面。
    ob_gzhandler() 在发送压缩数据前，它确定浏览器支持的编码类型 ("gzip", "deflate" 或不是其中任何一个)，然后返回有效的内容。所有浏览器都可以被涉及，因为浏览器本身会发送头文件，其中标注着该浏览器所支持的页面类型。

    仅在 Php 4.05 (或>) 下工作，且Zlib 必须为激活状态 (命令phpinfo();)，且为Apache，且mod_gzip 为激活状态。


  • FileManager : 激活或关闭使用FileManager （文件管理器）。请注意，仅在您需要的时候才激活该FileManager。


  • cookie 的有效时效：无注释 (1年为8760小时)


  • 网站名称 :您的网站的名称（当然是由您来定义）。


  • Nom du site pour la balise Title : 每个HTML页面的抬头标题 (主要为搜索引擎使用)


  • 网站URL地址 : http://www.xxx.com 或 http://www.xxx.xom/~homepage/...


  • 网站的Logo : 为gif或jpg图片文件。请将其保存在您服务器下images文件夹中(默认设置)，或保存在您的某个主题下 (完整路径，例如：themes/mon_theme/..../logo.jpg)。
    该图片文件仅在NPDS转印过程中使用。建议使用黑白图片。


  • 网站的标语Slogan : 如果您的网站有口号宣传，就在这里填写吧！


  • 网站启动日期 : 该启动日期 (20/10/1429 或31 Décembre 2012) 将仅在统计数字中显示。


  • 页面Top x 的事件的数目: 允许设置参数Top x (5, 10, 15, ...).


  • 在主页上显示的文章的数目 : 在一个页面上显示的文章以10篇为适宜。但是这最终取决于您的网站本身的需要。请注意在NPDS中注册的会员可以修改这个数值，因此，这里定义的文章数目也仅为一个默认的初始值。


  • 旧文章库中的文章数目 : 页面右侧的功能框之一显示的为 旧文章的列表。您可以对其中的显示的文章数目设置参数。


  • 激活 Export-News : Export-News是2个文本文件，保存在您网站的隐藏目录下。默认的参数为：否。
    在这2个文本文件中 (ultramode.txt 和 net2zone.txt) 包含您的网站的10个最新消息的内容。
    该文件允许其它网站引用您的信息资源，格式简易 :
    -ultramode.txt 文本文件中，每条新闻之间以%%间隔
    -net2zone.txt 为XML格式（net2zone）的文本文件

    "Online"在线功能与 Backend 的功能是相同的。后者是RDF/XML格式，或者更好的NPDS Push Infos System功能。


  • 匿名者的默认名称 : 默认的尚未注册的用户的名称。其实，匿名者本身就是很好的称呼值（当然也可以是访客等等）。在NPDS运行期间请不要修改这个参数。


  • 允许匿名者评论 : 如果您期望匿名用户（即非注册的用户）可以发表评论，就在本项中选择 是 。


  • 用户的评论的最多数目 (24小时内) : 对于匿名或注册会员，本参数将限制其发表评论文章的最大数目。 默认值为6， 99999为最大值。


  • 允许创建"News" : 该功能允许 版主/管理员 限制（或不限制）来自会员的新闻的提交。
    该功能对于一个信息网站的维护很有用。


  • 在统计数据中，无需保存“访问最多的的作者”的信息 : 旨在不将您的网站的维护者统计为真正的访问者。


  • 网站默认的显示主题 : 您的网站默认选择的主题。请不要忘记：注册用户可以修改本选项 (如果有多个主题可选择...)。

  • 启动页面 : 旨在设置启动页面（即您的网站的首页）的参数。系统默认的首页为index.php。但是您可以选择其它功能：
    - links.php
    - faqs.php
    - static.php?op=npds.txt&npds=1 ... (npds的参数可以为 -1 : 无功能框, 0 : 左功能框, 1 : 左右功能框)

    - 编辑：
    => 仅 index.php?op=edito : 显示编辑功能，及新闻系统。
    => 仅 index.php?op=edito-nonews : 显示编辑功能，但是不显示新闻系统。
    => 仅 index.php?op=newindex
    => 仅 index.php?op=edito-newindex
    => 仅 index.php?op=newtopic&topic=X

  • 选择网站的语言 : 无注释。


  • 激活多语言功能 : 允许NPDS使用多种语言，标记为 [langue]...[/langue]
    请同样定义当地时间，例如：[french]fr_FR[/french] 或 [english]en_EN[/english] 或 [chinese]cn_CN[/chinese]


  • 当地时间 : 请校准您所在的时区设置。Unix用户请看"locales"功能，通常在 /usr/share/locale 目录下。


  • 设置白天开始于 和 晚上开始于：旨在规定某些功能（如编辑功能）开始于某特定时段。


  • GMT : 旨在与主机所在的时区保持某个专门时差 (+或-)。





    • Banner 设置选项

  • 激活网站上的banner条幅：您可以激活或关闭网站上的banner广告。但是您首先必须定义了一个宣传活动...


  • 您的IP地址，目的在于避免将来自您的页面点击统计在数据中 : 请尊重诚实的统计数据。作为网站的管理员，您的计算机应该排除在网站浏览量的统计对象之外。





    • 页尾的信息

  • 有1,2,3,4行 : 您的网站的页尾部分支持HTML。如果您仅需要一行或两行内容，则保留该区域为空（直接跨过）。 通常情况下，该页尾部分是保留给以下信息：版权，帐号，主机寄存处，等等。





    • Backend信息的设置 (RDF/XML格式的信息来源)

  • Backend的题目 : 无注释。您可以在题目后面加上网站标语。


  • Backend的语言 : 网站本身使用的语言格式。


  • Backend的图片的URL地址 : 完整路径 (http://...) 在您网站上引入一个 image/logo 图片。


  • Backend的图片的宽度 : 无注释。


  • Backend的图片的高度 : 无注释。





    • 友好链接的默认设置

  • 每页上的链接的数量：每页可显示的链接的最大数量 (友好链接的功能中)。


  • 定义升级为 热门链接 所必须的点击数：无注释。当条件满足时，在该链接右侧显示一个小星号。


  • '新'链接的数量：Le nombre maximun de liens dans la page "Nouveau".


  • '最佳'链接的数量：Le nombre maximun de liens dans la page "Populaire" (les liens les plus "cliqués").


  • 搜索结果中显示的链接数量：无注释。


  • 允许匿名用户推荐并张贴链接：是或否。


  • 在链接页面上显示Logo图片：是或否。


  • 激活 给某类别附加小图标 [新] [New]的功能：当某类别为新，或其中有新内容时，显示该小图标。





    • 邮件系统 (Email)

  • 管理员的E-mail地址：当NPDS自动发送Emails时，本设置允许收信人回复给您 (真实或虚拟Email地址 / 格式要求填写完整的域名。=> 例如 : developpeur@npds.org)。


  • 所使用的邮件功能：fonction : mail => 所有Linux机器中默认使用此称谓 / fonction : email => 适用于某些寄存主机。


  • Email信尾 : NPDS可以发送某些Email (例如密码通知)。该选项允许您设置此类Email的信尾（例如签名等）。该区域支持HTML。 建议：在信尾提示您的地址，电话或网站的标语等。


    • 通过Email发送新文章给管理员：NPDS(如同 PHP-Nuke)允许系统在用户发表新文章时立刻通知管理员。

  • 通过E-mail提示有新文章：是或否。


  • 发送通知消息时的接收E-mail地址：无注释。


  • E-mail主题：无注释。


  • E-mail的消息：无注释。


  • E-mail (来源)帐号：无注释。





    • 对评论文章的控制

  • 控制的类型：系统对评论的节制可以是自动的，但是请不要盲目地相信她的作用。

    • - 管理员控制
    - 用户控制
    - 无控制

  • 对评论文章的字数的限制：定义一个评论文章能拥有的最多字数，以预示出现与否 "读评论的更多内容..." 的链接。4096是一个比较理想的数值。





    • 网上调查的设置

  • 网上调查中可选项的最多数量：无注释。


  • 调查投票结果的杠形分级显示：设置时，您输入一个数字，系统将乘以100倍。然后以此为基础计算将显示的分级结果（杠形图形方式）。如果您输入3，分级为300%。


    • 在99,99%的情况下，采用1是较为理想的设置。
  • 允许用户多次投票：是或否，这取决于您的偏好。如果您定义为“是”，系统允许用户在24小时内随意投票，并在调查中张贴一个短小信息。如果您定义为“否”，则一个用户在一个调查中的投票是唯一的。


  • 激活对网上调查的评论：无注释。





    • 关于图形和样式

    不要更改此选项，除非....

  • 主题中的图片的路径：通常为 /images/topics.


  • 某些图片（投票等）的路径：通常为 /images/menu/.


  • 管理员菜单的图片的路径：通常为 /images/admin/.


  • 激活在管理菜单中的图形：是或否。选择否的情况下，页面速度提高。


  • 激活管理中的短菜单：是或否。选择是的情况下，页面速度提高。





    • 其它

  • 管理模式下的文章数目：无注释 (但是本数值同样用于栏目等)。


  • 用户密码设置的最少字数：建议定义为8。


  • 列表中用户的数量：进入下一页之前在通讯录中。


  • 激活表情符号：NPDS允许在创建会员或论坛中不使用表情符号。


  • 个性化表情符号的最大尺寸为 (宽度*高度 / 60*80，以pixel为单位)：NPDS允许 专门定义上传的表情符号的最大尺寸。


  • 激活用户的简短描述：NPDS允许一个新用户不输入其ICQ, MSN等信息。


  • 准许自动创建会员：NPDS允许，在无管理员意见的情况下，不同意一个新会员的连接（目的：准许用户的登记卡的连接）。本功能对“非开放式”的网站很实用。


  • 准许用户选择自己的密码：您可以准许一个新用户选择他自己期望的密码。您同样可以预先定义为不准许该功能。


  • 准许订购服务：准许您的会员订购NPDS上的不同信息资料 (主题，论坛，内部消息等)。他们将通过Email接收订阅的消息 (请注意本功能可以会导致较大的系统负载)。


  • 准许会员为隐身状态：准许您的会员为“隐身”：他们的标识将不显示在NPDS的各个功能中（但是管理员可以观察到隐身会员）。


  • 关闭新的注册：NPDS允许临时性地关闭新会员注册功能 (static/closed.txt页面将显示)。本功能对关闭某网站很实用。


  • 激活http参照：无注释。


  • 最多参照的数量限制：请不要让您的数据库负载过多。


  • 会员列表：私人：NPDS允许仅对注册会员显示通讯录。


  • 上传栏目：选择首先出现的下载栏目（或类别），或者不做选择。


  • 简短评论：将评论转化为留言本中的记录。





    • HTTP其它

  • 在有信息外来源(头条标题)的新闻站点中，激活在Port 80对某站点的核查：NPDS同样允许不对某站点核查Port 80。 这点对于Multimania和其它ISP很实用。


  • 在HTML页面，激活对缓存管理的标记语言：NPDS允许使用对HTML页面的缓存管理的先进功能。某些ISP (例如Multimania) 并不允许这一点。


  • 在某些帖子（如论坛中）中，激活DNS：NPDS允许“捕获”IP地址和DNS名称。请注意本选项可能导致服务器的CPU被大量占用 (甚至可能被禁止)。





    • 系统其它

  • SaveMysql的保存文件最大不应超过：对dump ASCII文件在压缩前的大小的限制。


  • SaveMysql的保存类型：采用保存时文件类型选择。


  • 激活Tiny-Mce编辑器：NPDS可以使用Tiny-Mce编辑器 (IE, Firefox, ...)
